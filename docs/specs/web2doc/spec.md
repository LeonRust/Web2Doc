## 1. Problem Statement（为什么做）

AI Coding 工具在生成代码时，倾向于依赖模型训练时的旧知识，对快速演进的库/框架常给出**过时或错误**的 API 用法。开发者缺少一种低成本方式，把某个库/框架的**官方最新文档**整体转化为可直接喂给 AI 的本地上下文，用以**约束 AI 按权威文档而非记忆作答**。

Web2Doc 解决：给定一个在线文档站首页，自动抓取**整站文档正文**（跨多页/多链接），去除非正文噪声，转为结构化本地 Markdown（含本地化的图片资源），形成可离线、可作为 AI 上下文的文档集。

---

## 2. Success Metrics（可测试的成功标准）

针对一个典型的服务端渲染/静态文档站（具备浏览器时亦适用于动态站），运行一次后：

- **S1 覆盖率**（分母精确定义见 §2.1）：在**未触发 max-pages 截断**时，发现并抓取的文档页面数 ≥ 基准全集的 **95%**；触发截断时按 §2.1 的 Partial 口径判定。实际覆盖率（已抓页数 / 基准全集）须被度量并在运行报告中给出。
- **S2 去噪**：每页产物正文符合 §3 的"正文边界定义"——保留项齐全、丢弃项为 0（抽检 ≥10 页）。
- **S3 图片本地化**：正文中引用的图片本地化成功率 ≥ **95%**，且 Markdown 中图片链接均指向存在的本地相对路径，**无死链**。
- **S4 视频/嵌入保真**：视频与第三方嵌入在产物中**保留可访问的原始链接**（不产生死链、不丢失）。
- **S5 结构可用**：产物目录层级**镜像源站 URL 路径**；存在覆盖全部已抓页面的总索引；每个 `.md` 均为合法 Markdown，离线可读。
- **S6 确定性与续传**：抓取在给定最大页数上限内**确定性停止**；中断后重跑可断点续传，**不重复下载**已完成的页面与资源，**亦不产生半成品页面**（已记为完成的页面其 `.md` 必已写出）。
- **S7 成本可控**：LLM 调用次数为**站点级别（与页面数量无关，不随页面线性增长）**。
- **S8 降级可用**：在无浏览器环境下仍能完成静态站点抓取；遇到疑似动态渲染站点抓取不全时给出**明确告警**。
- **S9 失败口径**：成功率 = 成功页 / **已发现并入队的页面数**（分母**不含** discover 阶段未发现的页面）。**成功率 ≥ 80% 视为整次成功**（失败页计入告警清单），**< 80% 视为整次失败**（以非零状态结束并输出失败清单）。阈值可配置。**S1（覆盖率）与 S9（抓取失败率）是相互独立的判据：任一不达标即整体不达标，不得互相掩盖**（覆盖率不足不得因抓取成功率高而被忽略；**因 max-pages 主动截断导致的 Partial 不计为覆盖率失败**，见 §2.1）。

### 2.1 基准全集与覆盖率口径（S1 分母定义）

- **基准全集** = 同时满足以下全部条件的页面 URL 集合：
  1. 落在抓取**前缀**内（C3）；
  2. **robots 允许**（除非显式忽略，C9）；
  3. **判定为文档正文页**——排除门户首页索引、标签/分类/作者页、变更日志、搜索结果页、404/错误模板等非正文页。
  来源按降级链确定：`sitemap.xml` → 渲染后导航全集 → 前缀内可达集（D1）。
- **覆盖率判定**：
  - 当 `基准全集大小 ≤ max-pages`：实际抓取页数须 ≥ 基准全集 95%，否则**整体不达标**。
  - 当 `基准全集大小 > max-pages`（用户主动上限导致截断）：标记 **Partial（部分抓取）**，**不据此判失败**；但运行报告**必须透明给出**基准全集大小、上限值、未覆盖页数，及"提高 --max-pages 获取完整"提示。

## 3. 正文边界定义（去噪的 WHAT 口径）

界定"何为正文"，作为 S2 的可验证依据。

- **必须保留**：页面主标题（H1）、正文段落、列表、代码块、表格（含 API 参数表）、提示框/警告框（Admonition）、正文内联链接、正文中的图片。
- **必须丢弃**：全站顶部导航栏、侧边目录树、页脚、广告、Cookie/同意横幅、面包屑、"上一页/下一页"翻页器、"编辑此页 / 在 GitHub 编辑"、"最后更新于…"、版权与社交分享条、版本/语言切换器、站内搜索框。
- **链接改写**：指向**已抓取本地页面**的链接改写为本地相对路径；指向站外或未抓取页面的链接**绝对化后保留**（相对 URL 必须补全为绝对地址）。
- **资源绝对化**：所有**未成功本地化**的图片、视频与第三方嵌入，其地址一律绝对化保留，避免离线打开时出现死链（支撑 C7）。

## 4. User Stories（谁在什么场景用）

- **U1**：作为使用 AI Coding 工具的开发者，我想用一条命令把某框架官方文档抓成本地 Markdown，以便作为上下文喂给 AI，避免它用过时知识写错代码。
- **U2**：作为开发者，我只想提供首页 URL，工具就自动跨页抓取整站文档，无需逐页手动保存。
- **U3**：作为未安装浏览器的用户，我希望工具仍能处理静态文档站；若目标是动态站点导致内容缺失，我希望得到清晰提示而不是静默产出空文档。
- **U4**：作为重复使用者，我希望中断后能续传、重跑不浪费已完成的工作，且不会得到写了一半的页面。
- **U5**：作为需要整体投喂的用户，我希望可选地把全部正文合并为单一文件，便于一次性塞入上下文窗口。

## 5. Acceptance Criteria（怎么验证）

- **A1**：给定某静态/SSR 文档站首页 URL 执行，产出一个本地目录，包含镜像路径的 `.md` 文件、一个总索引、一个进度文件，且**运行报告给出覆盖率统计**，满足 S1、S5。
- **A2**：抽检若干产出页，正文符合 §3 边界定义（S2）；正文内图片均已下载到本地资源目录且链接可用（S3）；视频/嵌入保留原链接（S4）。
- **A3**：设置较小的最大页数上限执行，抓取在该上限内停止（S6）；中断后重跑，已完成页面/资源不被重复处理，且不存在"标记完成却无 `.md`"的半成品（S6）。
- **A4**：在未提供浏览器能力的环境执行，静态站点正常产出；对动态站点产出中触发并显示缺失告警（S8）。
- **A5**：统计一次运行的 LLM 调用次数，验证其不随页面数增长（S7）。
- **A6**：在未提供 LLM 凭据/网络的环境执行，工具不致命退出，使用回退规则继续产出（支撑 S8/可用性）。
- **A7**：构造含部分坏链的站点，验证失败页被记录、整次成败按 80% 阈值判定（S9）；并验证覆盖率不足（非截断所致）时即使抓取成功率高仍判整体不达标（S1×S9 独立性）。
- **A8**：开启合并产物选项时，额外生成一个包含全部已抓正文的单一文件；默认关闭时不生成（C8、U5）。
- **A9**：对设置了 `robots.txt` 抓取限制的站点，默认不抓取被禁路径；启用忽略开关后方可抓取（C9）。
- **A10**：当基准全集 > max-pages 时，运行标记 **Partial** 且**不判失败**，报告透明给出未覆盖页数与"提高上限"提示（§2.1）。

## 6. Non-Goals（v1 明确不做，划清边界）

- **N1**：不提供全文搜索 / 检索服务（仅产出文件）。
- **N2**：不做定时同步 / 增量订阅 / 版本 diff（断点续传属容错，非订阅）。
- **N3**：不下载视频文件（仅保留链接）。
- **N4**：不绕过登录、付费墙、验证码或任何访问控制。
- **N5**：不保证在无浏览器环境下完整抓取需 JS 渲染的站点。
- **N6**：不输出 Markdown 以外的格式（不产出 PDF / HTML / EPUB）。
- **N7**：不对内容做翻译、摘要或改写（保持原文）。
- **N8**：不做多站点批量编排（v1 为单站点单次运行）。

---

## 7. Constraints（外部约束，非实现方案）

- **C1**：须能处理需 JS 动态渲染的站点（在具备浏览器渲染能力时）；不具备时降级为静态抓取并告警。
- **C2**：依赖外部 LLM 服务，须兼容 OpenAI 风格的 Chat Completions 接口；服务端点、模型、密钥均可配置，密钥经环境变量/配置注入（见 constitution §5）。
- **C3**：抓取范围默认限定在首页 URL 的**路径前缀目录**内（例：首页 `…/docs/intro` → 前缀 `…/docs/`）；前缀可被显式覆盖或追加；默认**不**自动扩散到同域其它前缀。并受**最大页数上限**约束。
- **C4**：须对目标站点保持礼貌：并发数与请求间隔可配置，默认保守；不绕过访问控制。
- **C5**：运行环境可能无浏览器、无 LLM 凭据或网络受限，须有相应降级行为。
- **C6**：跨平台可用，macOS 优先。
- **C7**：产物须可离线阅读、自洽（相对链接、本地资源），可直接作为 AI 上下文。
- **C8**：默认仅产出"分页 + 总索引"；提供**可选**的全文合并产物，默认关闭（不违背 N1/N6，仍为 Markdown）。
- **C9**：默认**尊重**目标站点 `robots.txt` 的抓取限制；提供显式开关在用户知情时忽略（忽略时仍受前缀与最大页数约束）。
